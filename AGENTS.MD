# AGENTS.MD

## 프로젝트 목적

이 프로젝트는 세븐스플릿 알고리즘을 기반으로 자동매매 시스템을 만드는 프로젝트다.

목표는 사용자가 사전에 정한 가격 밴드, 슬롯 수, 자금 배분, 목표 수익률에 따라 매수와 매도를 자동으로 판단하고 실행하는 것이다. 시스템은 예측 모델이 아니라 규칙 기반 실행 엔진으로 동작해야 한다.

자동매매 시스템은 최소한 다음 흐름을 지원해야 한다.

- 전략 설정: 상단 가격, 하단 가격, 슬롯 수, 총 투자금, 목표 수익률, 손절 또는 대기 정책을 입력한다.
- 슬롯 생성: 설정값을 기준으로 7개 슬롯의 매수가, 목표 매도가, 투입 금액을 계산한다.
- 시세 감시: 현재가를 주기적으로 가져와 각 슬롯의 매수/매도 조건을 평가한다.
- 주문 실행: 조건을 만족한 슬롯에 대해서만 매수 또는 매도 주문을 실행한다.
- 상태 저장: 슬롯 상태, 주문 결과, 체결가, 수량, 시각을 영속적으로 저장한다.
- 복구: 프로세스 재시작 후에도 저장된 상태를 기준으로 이어서 판단한다.
- 로그와 감사 추적: 모든 판단, 주문, 실패, 보류 이유를 기록한다.

실거래 연동 전에는 반드시 모의 실행, 백테스트, 또는 페이퍼 트레이딩 모드로 검증할 수 있어야 한다.

## 스플릿 투자 방식

이 프로젝트의 핵심 전략은 스플릿 투자 방식이다.

스플릿 투자는 투자금을 여러 조각으로 나누어 한 번에 진입하지 않고, 가격 변화에 따라 단계적으로 매수하고 각 매수 조각을 독립적으로 매도하는 전략이다.

핵심은 예측이 아니라 대응이다. 바닥과 고점을 맞히려 하지 않고, 미리 정한 규칙에 따라 자금 투입과 수익 실현을 반복한다.

## 기본 구조

- 전체 투자금을 여러 개의 슬라이스로 나눈다.
- 보통 5~10개 단위로 나누며, 대표적으로 7분할 구조를 사용할 수 있다.
- 사전에 정한 가격 조건에 도달한 슬라이스만 매수한다.
- 가격이 하락하면 정해진 간격마다 다음 슬라이스를 추가 매수한다.
- 가격이 상승하면 수익 조건을 만족한 슬라이스만 매도한다.
- 각 슬라이스는 독립적인 포지션처럼 관리한다.

## 세븐스플릿 알고리즘: 가격 범위 사전 설정 방식

세븐스플릿은 시작 전에 상단 가격과 하단 가격을 정하고, 그 범위 안에 7개의 독립 슬롯을 배치해 운영한다.

### 사전 설정

운영 시작 전 다음 값을 한 번 설정한다.

- 상단 가격: 1번 슬롯의 매수 기준가
- 하단 가격: 7번 슬롯의 매수 기준가
- 슬롯 개수: 7개
- 슬롯 간격: `(상단 가격 - 하단 가격) / 6`
- 분할 방식: 균등 가격 간격 또는 비율 기준 간격
- 슬롯당 투입 금액: `총 투자금 / 7`
- 목표 수익률: 전체 슬롯 공통 또는 슬롯별 차등 적용

슬롯별 매수가는 다음과 같이 만든다.

- 1번 슬롯 매수가: 상단 가격
- 2번 슬롯 매수가: 상단 가격 - 슬롯 간격
- 3번 슬롯 매수가: 상단 가격 - 슬롯 간격 * 2
- 4번 슬롯 매수가: 상단 가격 - 슬롯 간격 * 3
- 5번 슬롯 매수가: 상단 가격 - 슬롯 간격 * 4
- 6번 슬롯 매수가: 상단 가격 - 슬롯 간격 * 5
- 7번 슬롯 매수가: 하단 가격

각 슬롯의 목표 매도가는 기본적으로 다음 공식으로 계산한다.

```text
목표 매도가 = 슬롯 매수가 * (1 + 목표 수익률)
```

단, 낮은 가격대의 슬롯일수록 더 높은 목표 수익률을 적용하는 방식도 허용한다.

### 슬롯 상태

각 슬롯은 다음 상태 중 하나를 가진다.

- `EMPTY`: 비어 있음, 매수 대기
- `HOLDING`: 보유 중, 매도 대기

### 매수 조건

각 슬롯은 독립적으로 매수 여부를 판단한다.

- 슬롯 상태가 `EMPTY`이고
- 현재가가 해당 슬롯의 매수가 이하로 내려오면
- 해당 슬롯을 매수하고 상태를 `HOLDING`으로 변경한다.

가격이 갭하락해 여러 슬롯 조건이 동시에 충족되면, 조건을 만족한 모든 `EMPTY` 슬롯을 매수한다.

### 매도 조건

각 보유 슬롯은 독립적으로 매도 여부를 판단한다.

- 슬롯 상태가 `HOLDING`이고
- 현재가가 해당 슬롯의 목표 매도가 이상으로 올라오면
- 해당 슬롯을 매도하고 상태를 `EMPTY`로 변경한다.

### 재매수 규칙

매도된 슬롯은 다시 `EMPTY` 상태가 된다. 이후 가격이 다시 해당 슬롯의 매수가 이하로 내려오면 같은 규칙에 따라 재매수한다.

### 범위 이탈 처리

상단 가격을 이탈한 경우:

- 현재가가 1번 슬롯 매수가보다 위에 있으면 신규 매수하지 않는다.
- 보유 슬롯이 있다면 각 슬롯의 목표가 도달 여부만 판단한다.
- 모든 슬롯이 비어 있다면 재진입하지 않고 대기한다.

하단 가격을 이탈한 경우:

- 현재가가 7번 슬롯 매수가보다 아래에 있으면 7개 슬롯이 모두 매수된 상태로 간주될 수 있다.
- 추가 매수는 하지 않는다.
- 사전에 정의한 손절 규칙을 실행하거나 추가 대기한다.

### 밴드 재설정

다음 상황에서는 밴드를 재설정하거나 운영을 중단한 뒤 재시작할 수 있다.

- 가격이 상단을 크게 돌파한 뒤 일정 기간 머무는 경우: 밴드를 위로 이동
- 가격이 하단을 크게 이탈한 뒤 일정 기간 머무는 경우: 손절 후 새 밴드 설정
- 금리 사이클 전환 등 거시 환경이 크게 바뀐 경우

### 핵심 데이터 구조

전체 운영 정보는 최소한 다음 값을 가진다.

- 상단 가격
- 하단 가격
- 슬롯 개수
- 슬롯당 자금
- 목표 수익률

각 슬롯 정보는 최소한 다음 값을 가진다.

- 슬롯 번호
- 매수 예정가
- 목표 매도가
- 현재 상태
- 실제 매수가
- 보유 수량
- 매수 시각

## 슬라이스 단위 관리

각 슬라이스는 최소한 다음 정보를 가진다.

- 진입 가격
- 수량
- 투입 금액
- 목표 매도 가격
- 현재 상태: 대기, 보유, 매도 완료

전체 포지션의 평균 단가만 보고 판단하지 않는다. 매수된 각 슬라이스가 자기 기준으로 목표 수익에 도달했는지를 본다.

## 매수 원칙

- 한 번에 모든 자금을 투입하지 않는다.
- 다음 슬라이스 매수 조건은 사전에 정한다.
- 하락했다고 무조건 매수하지 않는다.
- 최대 슬라이스 수에 도달하면 추가 매수하지 않는다.
- 남은 슬라이스를 위한 현금을 항상 보존한다.

## 매도 원칙

- 매도는 슬라이스별로 판단한다.
- 특정 슬라이스가 목표 수익률에 도달하면 해당 슬라이스만 매도한다.
- 일부 수익 실현 후에도 나머지 슬라이스는 기존 규칙에 따라 유지한다.
- 수익 계산에는 수수료, 세금, 슬리피지, 스프레드를 반영한다.

## 리스크 관점

스플릿 투자는 물타기와 비슷해 보일 수 있지만, 핵심 차이는 총 투입 한도와 매수 규칙이 사전에 정해져 있다는 점이다.

이 전략은 회복 가능성이 높고 유동성이 충분한 자산에 더 적합하다. 예를 들면 주요 ETF, 대형 우량주, 달러, 금 같은 자산이다.

반대로 상장폐지 위험, 재무 부실, 유동성 부족, 구조적 하락 가능성이 큰 자산에는 부적합하다.

## 자동매매 시스템에서 중요한 점

- 모든 매수와 매도는 명시적인 규칙으로만 실행한다.
- 슬라이스 상태는 주문 전후에 반드시 저장한다.
- 시스템 재시작 후에도 같은 상태에서 이어서 판단할 수 있어야 한다.
- 수동 주문이나 부분 체결이 발생해도 슬라이스 회계가 깨지면 안 된다.
- 봇은 매수, 매도, 보류, 중단의 이유를 로그로 남겨야 한다.

## 구현 원칙

- 전략 계산 로직과 거래소/증권사 API 연동 로직을 분리한다.
- 매수/매도 판단 함수는 같은 입력에 대해 항상 같은 결과를 내는 순수한 로직에 가깝게 유지한다.
- 실제 주문 실행 전에는 주문 가능 금액, 최소 주문 단위, 수수료, 세금, 슬리피지, 호가 단위를 검증한다.
- 주문 요청과 실제 체결은 다를 수 있으므로 주문 상태와 체결 상태를 분리해서 관리한다.
- 부분 체결, 주문 실패, 취소, 재시도, 네트워크 오류를 정상적인 상태 전이로 다룬다.
- 같은 슬롯에서 중복 주문이 발생하지 않도록 멱등성 키나 주문 잠금 장치를 둔다.
- 실거래 모드와 모의 거래 모드는 같은 전략 엔진을 사용하되 주문 실행 어댑터만 다르게 구성한다.
- API 키, 계좌번호, 토큰 같은 민감 정보는 코드나 문서에 직접 저장하지 않는다.
- 기본값은 보수적으로 둔다. 불확실한 경우 매수/매도보다 보류와 로그 기록을 우선한다.

## 권장 모듈 경계

구현 시 다음 책임을 분리하는 것을 우선한다.

- `strategy`: 세븐스플릿 슬롯 생성, 매수/매도 조건 평가, 밴드 재설정 판단
- `portfolio`: 슬롯 상태, 보유 수량, 평균 체결가, 현금, 실현 손익 관리
- `broker`: 거래소/증권사 API 연동, 주문 요청, 주문 조회, 체결 조회
- `market_data`: 현재가, 호가, 캔들, 환율 등 시세 데이터 조회
- `storage`: 전략 설정, 슬롯 상태, 주문/체결 이력 영속화
- `runner`: 주기 실행, 이벤트 루프, 장애 복구, 모드 전환
- `risk`: 손절, 최대 투자금, 종목 제한, 거래 중단 조건 검증
- `logging`: 판단 근거, 주문 결과, 예외 상황 기록
